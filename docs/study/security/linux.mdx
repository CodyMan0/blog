## 본론

### 리눅수가 무엇인가?

24년 5월 기준, 개인 사용자 점유율 4퍼센트 밖에 안되지만 서버는 62% 점유중. 유닉스에 뿌리를 두고 있기에 리눅스 익히면 다른 OS의 사용법도 수월하게 익힐 수 있음.

- 리눅스 오픈소스, 자유롭게 코드 변경하여 사용 가능 그래서 배포판 존재 (Ubuntu, CentOs, Fedora ...)

### Shell이 무엇인가?

OS는 크게 2개 수준

- User Mode
- Kernel Mode

유저모드와 커널 모드의 중간 다리 역할이 필요한데 이를 Shell이 담당한다. 일반 사용자가 커널과 소통하기 위한 프로그램이라고 생각하면 됨.
Shell을 자유자재로 다루는 게 OS를 잘 사용 할 수 있다는 것, 가장 기초적이면서 중요한 부분임.

Shell을 정리해보면 사용자와 OS 간의 인터페이스를 제공하는 소프트웨어라고 함.

Shell에는 2가지

- GUI
- CLI

논리적으로 GUI가 CPU,GPU 소모 많기에 실행 속도가 느리다는 것. CLI는 상대적으로 빠르고 정밀하다는 것.

#### 터미널

CLI Shell을 GUI 위에서 사용할 수 있도록 하는 프로그램.

1. sh : 최초 셸
2. bash : sh 대체하기 위해 작성된 Unix Shell
3. zsh : 유닉스 최초 셸,

> sh 대비 추가된 기능

- 명령 히스토리 지원
- Tab 자동 완성 지원

### 파일과 디렉터리

리눅스, hardware 장치, 소켓, 프로세스 정보 같은 시스템 자원들을 모두 파일로 관리하고 디렉터리는 이러한 파일 체계적으로 구성

파일 : 리눅스에서는 확장명을 작성할 필요가 없음. 자동적으로 OS가 파일 시그니쳐를 보고 특정 바이드로 해당 파일 종류를 나타냄.
디렉터리 : 파일 분류를 위한 Namespace

### 프로세스 생성 및 관리

프로그램은 원래 저장장치에 저장된 파일 형태로 존재, 실행 명령으로 메모리로 불러와 작동한다. 이것을 프로세스라고 한다.

프로세스 구조와 역할을 살펴보고 사용자가 어떻게 제어할 수 있는지 학습!

프로세스란
컴퓨터에서 실행되고 있는 프로그램. 결국 OS가 실행 요청을 받아서 명령 받아 프로세스 생성함. 생성된 프로세스는 Central Processing Unit에서 실행되고 동시에 실행되도 OS가 순서 조절함.

프로세스 5가지 상태

1. Create: 생성중
2. Running : 실행
3. Ready : CPU 점유하기 위해 대기 상태
4. Blocked : 입출력 대기, 자원 접근 불사등의 이유로 CPU를 양도하고 선수 작업이 완료되기를 대체하기
5. Terminated : 프로세스가 종료되는 중.

리눅스 프로세스

- Process ID : 현재 프로세스 아이디
- Parent Process ID : 부모의 프로세스 아이디

리눅스에서는 `fork()`를 사용해 현재 프로세스 복제해서 새로운 프로세스 생성함. 최상위 프로세스가 유일하게 존재하는데 이를 init 프로세스라고 한다.

`fork()`를 사용하면 복제한 원본을 부모, 생성되는 프로세스 자식이라고 함.

`exec()` 사용하면 프로세스를 복제한 후 자식 프로세스의 내용을 다른 실행파일의 내용으로 덮어씌운다.

자식 프로세스보다 부모가 먼저 종료되면 고아 프로세스가 된다. 반대는 좀비 프로세스라고 함.

프로세스 확인 (ps, htop)

`ps`는 현재 실행중인 프로세스 상태 보여줌

포그라운드와 백그라운드

프로그램 실행 방법
ls, cat은 그저 리눅스 파일 시스템의 `/bin/ls`, `/usr/bin/cat`에 위치한 실행 파일이다.

리눅스는 Executable and Linkable Format (ELF) 형식의 실행 파일 지원. Windows에서 .exe와 .dll 리눅스에서 돌릴 수 없음.
실행 파일로 만드는 법 : GCC와 같은 컴파일러 이용해서 코드 -> 실행 파일로 바꾸는 것.
