## 1. 논리적으로 글쓰기

28p
피라미드를 만들어놓은게 마치 테이블이나 엔티티를 솎아내는 것과 비슷하게 보인다. grouping이라는 것이 있네?

29p
사고하고 기억하고 문제 해결을 하는 동의 모든 프로세스는 이런 그루핑과 요약의 사고 프로세스를 수반한다.

30P
이해하기 쉬운 글은 먼저 전체를 요약한 생각을 서술한 다음 개별적인 생각을 하나씩 설명한다.

36p
규칙 3가지 공유해주심.

1. 어떤 계층에 있는 메시지든 하위 계층의 메시지를 요약해야한다.
2. 그룹 내의 메시지는 항상 동일한 종류여야한다.
3. 그룹 내의 메시지는 항상 논리적 순서로 배열되어야 한다.

생각을 말로 표현하거나 글로 쓸 때 가장 중요한 것은 하위 그룹의 메시지에서 새로운 메시지를 도출하는 일.

38p
메시지가 제대로 그루핑이 되었는지 여부를 점검하는 가장 간단한 방법 -> 그룹 내의 메시지들을 동일한 사항을 나타내는 하나의 복수명사로 묶을 수 있는지 확인

39p
이해하기 쉬운 글쓰기의 핵심은 글을 쓰기 전에 먼저 전달하고자 하는 메시지를 피라미드 형태로 배열하고 앞에서 살펴본 세가지 규칙을 충족하는지 확인하면됨.

47p
생각을 그루핑할 땐, 연역 혹은 귀납

49P
피라미드를 만들기 전에 핵심 메세지는 독자의 모든 질문에 답변해야한다는 점이다.

## 2. 논리적으로 생각하기

147p
피라미드 원칙 중 세번째 규칙은 그룹 내의 생각을 항상 논리적 순서로 배열해야 한다는 것이다.

156p
결론을 도출할 때는 사전에 근거가 되는 프로세스를 명확하게 인식하고 있어야 이해하기 쉬운 문서를 작성할 수 있다.
-> 나의 생각: 그 말인 즉은 이해하기 쉬운 문서를 작성하려면 결론의 근거의 프로세스를 명확하게 저자가 이해하고 있어야하는구나!?
-> 그럼 근거의 프로세스는 뭔데?

## 정리하며

-"논리적"이라는 것이 결국 생각의 틀을 튼튼하게 하면서 질서있게 정렬하는 것 같다는 느낌을 받는다. 논리적이어야하는 이유는 합리적인 커뮤니케이션을 위함이고 더 나은 결정을 내릴 수 있게 되기 때문이다.

- 논리적 글쓰기를 잘해야 코딩을 잘할 수 있다는 의미를 깨달음...
