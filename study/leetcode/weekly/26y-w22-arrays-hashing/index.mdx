## Problems Solved

### 1. Two Sum — Easy

> Let me walk you through my approach. I used a hash map to store
> each number's complement as I iterate. For each element `n`, I check
> if `target - n` already exists in the map. This brings the time
> complexity down from O(n²) to O(n), at the cost of O(n) extra space.

```ts
function twoSum(nums: number[], target: number): number[] {
  const seen = new Map<number, number>();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (seen.has(complement)) return [seen.get(complement)!, i];
    seen.set(nums[i], i);
  }
  return [];
}
```

**Reflection (KR)**:
brute force로 시작했다가 해시맵으로 바꾸면서 *"공간을 사서 시간을 얻는다"*
는 trade-off를 처음 의식적으로 느꼈음.
