# 경력 기술서 작성 가이드

## 경력 기술서란?

이력서가 **"어디서 무엇을 했는가"**의 요약이라면, 경력 기술서는 **"어떻게, 왜, 어떤 결과를 냈는가"**의 상세 설명입니다.

## STAR 기법

| 단계 | 의미 | 질문 |
|---|---|---|
| **S** (Situation) | 상황/배경 | 어떤 프로젝트였고, 어떤 문제가 있었는가? |
| **T** (Task) | 과제/목표 | 내가 해결해야 할 구체적 목표는? |
| **A** (Action) | 행동 | 내가 직접 취한 구체적 행동과 기술적 선택은? |
| **R** (Result) | 결과/성과 | 정량적 수치로 표현된 결과는? |

## 작성 원칙 (면접왕 이형 조언 기반)

1. **성과(Result)를 제일 앞에** — 채용 담당자는 빠르게 스캔한다
2. **"우리가" 아닌 "내가"** — 팀 프로젝트에서 내 구체적 기여도를 명확히
3. **당연한 역할에서 경쟁력 발굴** — 나만의 접근법과 결과를 부각
4. **가독성 최우선** — 30초 안에 핵심 파악 가능한 구조
5. **면접과 연결** — 경력 기술서 내용이 면접 답변과 일관된 스토리

## 나쁜 예 vs 좋은 예

```
❌ React를 사용하여 웹 서비스 개발

✅ 월간 리포트 수동 작성에 480분이 소요되고 고객 응대 30건이
   발생하는 문제를 인지 (S)
   → 리포트 자동화 기능 기획 및 구현 (T/A)
   → 리포트 생성시간 480분→2분(99.6%↓),
     고객 응대 30건→12건(60%↓) (R)
```
